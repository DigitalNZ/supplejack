## Sets
The User Sets in Supplejack API are groups of favourites that a user can publish. User Sets can return a list of sets from a user account, as well as returning a list of items for a specified set. Private API functions also exist to create, update and destroy sets, as well as add and remove individual records from a set.

#### Create a new set

| Parameters | Description |
| ------------ | ------------- |
| name | **required**  |
| description | **optional**  |
| privacy | **optional**, default is *public*. <br/> *public* - This set is public and it may appear in Supplejack search results.<br/>*private* - This set is private. Only you can see this set. <br/>*hidden* - This set is hidden. Only people you share this link with can see this set.|
| tags | **optional**, Array   |

Example request:

```ruby
POST: http://localhost:3000/sets.json?api_key={YOUR_API_KEY}
```

Parameters:

```ruby
{
    "set": {
        "name": "Dogs and cats",
        "description": "Ugly dogs and cats",
        "privacy": "hidden",
        "tags": ["cats", "dogs"],
        "featured": false        
    }
}
```

#### Update a set

| Parameters | Description |
| ------------ | ------------- |
| name | **required**  |
| description | **optional**  |
| privacy | **optional**, default is *public*. <br/> *public* - This set is public and it may appear in Supplejack search results.<br/>*private* - This set is private. Only you can see this set. <br/>*hidden* - This set is hidden. Only people you share this link with can see this set.|
| tags | **optional**, Array   |
| featured | **optional**, default is *false* <br/> You can set the `featured` parameter to `true` if you have an API key with admin role. Otherwise, passing `featured` parameter won't have an effect. |
| approved | **optional**, default is *false*, A flag for to determine approved public sets to be public. You should have an API key with admin role. |
| records | **optional**, Hash of records to add <br/> *record_id* - Integer, Id of a record <br/> *position* - Integer, position of the record inside the set. You can specify different position value on each record to modify the order. |

Example request:

```ruby
PUT: http://localhost:3000/sets/{SETS_ID}.json?api_key={YOUR_API_KEY}
```

Parameters:

```ruby
{
    "set": {
        "name": "Pretty Dogs and cats",
        "description": "Pretty dogs and cats",
        "privacy": "public",
        "tags": ["cats", "dogs", "pretty"],
        "featured": false,
        "approved": false,
        "records": [
            {
                "record_id": 78,
                "position": 1
            },
            {
                "record_id": 79,
                "position": 2
            },
            {
                "record_id": 80,
                "position": 3
            }
        ]        
    }
}
```

#### Delete a set

Deletes a specific set.

Example request:

```ruby
DELETE: http://localhost:3000/sets/{SETS_ID}.json?api_key={YOUR_API_KEY}
```

## Set Items
#### Add an item

Adds a new record/item to a set.

| Parameters | Description |
| ------------ | ------------- |
| id | **required**  |
| record | **required**, Hash<br/> *record_id* - Integer, Id of a record <br/> *position* - Integer, position of the record inside the set |

Example request:

```ruby
PUT: http://localhost:3000/sets/{SETS_ID}.json?api_key={YOUR_API_KEY}
```

Parameter:

```json
{
    "record": {
        "record_id": 90,
        "position": 1
    }
}
```

#### Delete an item

Example request:

```ruby
DELETE: http://localhost:3000/sets/{SETS_ID}/records/{RECORD_ID}?api_key={YOUR_API_KEY}
```
